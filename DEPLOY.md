# 布丁喵上线指南 🍮

三步就能让顾客手机下单、店里 POS 实时收单。跟着做即可，不需要懂代码。

---

## 第 1 步：建数据库（一次性，约 2 分钟）

1. 打开你的 Supabase 项目 → 左侧 **SQL Editor** → **New query**
2. 打开本仓库的 **`supabase-setup.sql`**，把里面**全部内容**复制粘贴进去
3. 点右下角 **Run**（跑成功会显示 Success）

这一步会自动建好 `orders`（订单）和 `menu_items`（菜单）两张表、开启实时推送、并把当前 13 个菜品灌进去。

> ⚠️ 这个脚本会**清空**已有订单再重建。只在第一次上线时跑；以后不要重复跑第 1 节，否则历史订单会没。

密钥说明：`pudding-meow.html` 和 `pos.html` 顶部已经填好了你的 `SUPABASE_URL` 和 `SUPABASE_KEY`（两个文件必须一致）。以后换项目时，两个文件都要改。

## 第 2 步：把网站放上线（GitHub Pages，免费）

前提：先把本次改动的 PR 合并到 `main` 分支（在 GitHub 点 Merge）。

1. GitHub 仓库 → **Settings** → 左侧 **Pages**
2. **Source** 选 **Deploy from a branch** → 分支选 **main** → 目录 **/(root)** → **Save**
3. 等 1–2 分钟，你会得到两个网址：
   - 顾客点单：`https://lim3232333-cmyk.github.io/pudding-meow/pudding-meow.html`
   - 店里 POS：`https://lim3232333-cmyk.github.io/pudding-meow/pos.html`

> 两个页面同源，实时联动天然成立。想要更短的自定义域名（如 `puddingmeow.com`）可以后面再加，告诉我即可。

## 上线前自测清单（在真实浏览器里，2 台设备或 2 个不同浏览器）

> 关键：要用**两台设备**或**两个不同浏览器**（例如手机 + 电脑），这样才证明是「云端跨设备」而不是本机。

1. **手机**打开顾客点单页 → 选商品 → 购物车 → 选「柜台支付」→ 确认下单
2. **电脑**打开 POS → 顶部「Pending Payment / 待付款」→ 应**几秒内**看到这单，标注 `📱线上·自取`，弹出 `🔔 新线上订单`
3. POS 点该单「已付款」→ **手机**上这单状态应实时变成「制作中」
4. POS 后台 → **菜单管理** → 改某个菜的价格 / 点「下架」→ **手机**刷新点单页 → 价格更新 / 该菜消失
5. TNG 预付单：顾客选 TNG 付款的单**不会**进待付款队列（因为已经线上付了）——这是正确的

全部通过 = 正式可营业。任何一步不对，把现象告诉我。

## 日常使用

- **改菜单 / 改价 / 上下架 / 加新品**：POS → 登录（默认 PIN `0000`）→ 进后台 → **菜单管理**。改完顾客端自动更新。
- **收单**：顾客线上下单进「待付款」；到店/柜台付款后点「已付款」。店员在 POS 自己开的单照旧。

## 收款方式

- **柜台/现金**：顾客下单，到店付款，店员点「已付款」。
- **TNG**：顾客下单时跳转到你的 TNG 收款链接。当前用的是代码里预置的链接——要换成你自己的 TNG 商家收款码/链接，告诉我即可替换。

## 已知限制（可后续升级）

- 顾客端「订单」页目前显示全部近期订单（没有「只看自己的」——那需要加会员登录识别身份）。
- 菜单后台写权限目前对匿名开放（POS 只有本机 PIN，没有真正的店员登录）。稳定运营后建议加店员账号鉴权收紧。
- 会员积分体系仍存在本机（未上云）。
